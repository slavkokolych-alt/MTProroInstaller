# MTProroInstaller
Установка своих прокси MTProto на свой сервер
# Инструкция: установка прокси и вход в дашборд

Для администратора без глубокого опыта с Linux и Docker. Текст описывает ваш стек: **Telemt** (MTProto-прокси), **скрипт/bootstrap**, **сервер мониторинга** с **Grafana**.

---

## 1. Что нужно заранее

1. **VPS** с **Debian или Ubuntu** с выделенным IP, доступ **root** по SSH (логин `root` и пароль или ключ — как даст хостинг).
2. **Доменное имя** для прокси (например `s4.neo-trading.org`), которым будут пользоваться в Telegram:
3. В панели DNS у регистратора домена: **A-запись** — имя поддомена → **IP вашего VPS** (после покупки сервера IP будет в панели хостинга):

<img width="774" height="439" alt="image" src="https://github.com/user-attachments/assets/cc86214e-0c82-422f-b71b-4702aa97cb81" />

4. Данные для **сервера мониторинга** (IP, логин SSH, пароль) — выдаёт хостер.
5. На своём **Windows-ПК**: файл **`TelemtBootstrap.exe`** (или установленный Python и скрипт `bootstrap_telemt_proxy.py` — как принято у вас в команде).

Логин и пароль для **входа в дашбоард** не храните в открытом доступе: их выдаёт владелец; в репозитории проекта они могут быть в файле `reports_access.txt` (этот файл не публиковать и не коммитить в git).

---

## 2. Подготовка файла настроек

1. Скопируйте пример **`telemt_bootstrap.example.json`** в **`dist/telemt_bootstrap.json`** (рядом с `TelemtBootstrap.exe` — обычно папка `dist`).
2. Откройте `telemt_bootstrap.json` в блокноте и заполните:

| Поле | Что указать |
|------|-------------|
| `host` | IP-адрес **нового** VPS прокси |
| `user` | Обычно `root` |
| `password` | Пароль SSH от root |
| `public_host` | Домен прокси **точно как в DNS** (например `s4.domain.org`) |
| `proxy_alias` | Короткое имя для мониторинга: **уникальное** среди всех прокси (`s1`, `s2`, `s3` … не повторять) |
| `monitoring_host` | IP сервера мониторинга |
| `monitoring_user` | Обычно `root` |
| `monitoring_password` | Пароль SSH мониторинга |

Сохраните файл в кодировке **UTF-8**.

---

## 3. Запуск установки прокси (новый сервер)

1. Убедитесь, что **DNS уже указывает** на IP сервера (иногда распространение занимает от нескольких минут до пары часов).
2. Запустите **`TelemtBootstrap.exe`** двойным щелчком (или из консоли), **из папки, где лежит** `telemt_bootstrap.json` (часто это `dist`).

Скрипт сам:

- поставит Docker при необходимости;
- запустит **Telemt** и **nginx** для порта **443**;
- выведет в окно и в лог **`telemt_bootstrap.log`** секрет для бота и ссылки.

3. Скрипт **остановится** и попросит **тег от @MTProxybot**:

   - В Telegram откройте **@MTProxybot**.
   - Команда **`/newproxy`**, затем отправьте боту строку вида **`s4.domain.com:443`** (домен и порт **443**).
   - Скопируйте из конфига на сервере **ровно 32 символа** секрета пользователя (в логе это явно написано).
   - Бот вернёт **TAG** (32 hex) — **вставьте его в окно скрипта** и нажмите Enter.

4. Дождитесь окончания: появятся строки про **туннель мониторинга**, **Grafana** и ссылки **`tg://t.me/proxy?...`** (обязательно сохраните эти ссылки в "Избранном" на вашем аккаунте).

5. **Проверка в Telegram:** Нажмите на ссылку. Прокси запустяться.

---

## 4. Как зайти в дашборд

1. Откройте в браузере сайт отчётов (у вас в проекте указан как **`http://reports.neo-trading.org/proxy-admin/`** — если владелец даст другой адрес, используйте его).
2. Браузер запросит **логин и пароль** (Basic Auth) — возьмите у владельца (не пересылайте посторонним).
3. После входа добавьте в соответствующие поля TAG из бота @MTProxybot и метку (любой текст - s4, IP сервера и т.п.):

<img width="939" height="201" alt="image" src="https://github.com/user-attachments/assets/c4686a37-c632-4d38-a004-5709cdd5b0ce" />

4. Нажмите кнопку **Добавить и перезагрузить Prometheus**.
5. Повторить если у вас несколько сервреров с прокси
6. Нажмите на **дашбоард Telemet Proxy**
6. Вверху выберите переменную **«Прокси»** (`s1`, `s2`, …) или **All**, чтобы смотреть метрики нужного сервера.

Если страница не открывается — проверьте VPN/браузер и что адрес скопирован **полностью** (с `http://` или `https://`).

## 5. Частые проблемы

| Симптом | Что проверить |
|--------|----------------|
| В Telegram «недоступен» | DNS A-запись на **тот же IP**, что VPS; на сервере открыт порт **443** (в панели хостинга firewall / security group: разрешить **443/tcp** с интернета). |
| Скрипт не подключается по SSH | IP, пароль, не блокирует ли провайдер SSH (порт 22). |
| В Grafana нет прокси в списке | Подождите 1–2 минуты после установки; проверьте, что **`proxy_alias`** уникален и туннель метрик в логе «работает». |
| «Unknown TLS SNI» / вечное «соединение» | На актуальных конфигах в `[censorship]` должны быть **`tls_domain = "ya.ru"`** и **`unknown_sni_action = "mask"`** — при старых серверах переустановите конфиг из свежего bootstrap или попросите того, кто ведёт репозиторий. |

---

## 6. Кому писать при ошибках

- **Сервер не пингуется, SSH не заходит** — сначала панель хостинга (сервер включён, IP не сменился, лимиты).
- **Логика скрипта, JSON, дашборд** — разработчик/владелец репозитория `neotrading`.
